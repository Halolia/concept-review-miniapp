# 角色权限矩阵

## 角色定义

| 角色 | 标识 | 说明 |
|------|------|------|
| **管理员** | `admin` | 系统管理者，拥有全部权限 |
| **评审专家** | `expert` | 被指派评审项目的专家 |
| **领导** | `leader` | 可查看汇总，不可操作 |
| **未绑定** | `guest` | 未在 users 集合中注册的用户 |

## 权限验证机制

**正式模式（DEBUG_MODE = false）**：

1. 每次操作调用云函数 `reviewFunctions`
2. 云函数获取 `OPENID` → 查询 `users` 集合
3. 验证 `status === 'active'`
4. 验证 `role` 在白名单中
5. 业务逻辑中二次验证（如：专家只能操作自己的指派）
6. 不允许前端传入 `role`、`expertId` 等敏感参数

**DEBUG 模式（DEBUG_MODE = true）**：

1. 所有操作由 `request.js` 的本地 `localFallback` 处理
2. 不调用云函数
3. 首页显示角色切换面板
4. 所有页面入口可见

## 安全原则

1. 前端权限控制仅作 UI 展示，不可作为安全依据
2. 所有关键权限在云函数中再次验证
3. 错误响应不返回完整 stack
4. 用户密码等敏感信息不通过云函数传参
5. OPENID 不在前端展示
